# ToolzAI Telegram Bot

An open-source Telegram digital-product store bot built on Cloudflare Workers and D1. This public repository contains the bot only; the private management website is maintained separately.

## Live bot

[Open @toolzai_bot on Telegram](https://telegram.me/toolzai_bot)

## Features

- Telegram command menu with `/start` for restarting the bot
- Human verification shown only until a user has successfully verified
- Product browsing, fixed BDT prices, balance purchases, order history, and warranty claims
- bKash, Nagad, and Upay balance deposits with exact-amount Firestore verification
- Rocket remains visible as temporarily unavailable until an account is configured
- Binance Pay (USDT) balance deposits using only the customer's Order ID
- Server-side Binance transaction lookup, received-amount crediting to two BDT decimal places, and one-time Order ID claims
- Temporary `Verifying...` feedback that is deleted after payment verification finishes
- Referral rewards, withdrawals, editable menus, announcements, and seller API integration
- Secure balance redeem codes generated by the private manager, with per-user protection, usage limits, and optional expiry
- Cloudflare D1 schema, ordered migrations, and automated tests

## Quick start

### 1. Install and configure

```bash
cd cloudflare-worker
npm install
npx wrangler login
npx wrangler d1 create digital_product_bot
```

Copy `wrangler.example.toml` to `wrangler.toml`. Replace every placeholder with your own non-secret configuration. The real `wrangler.toml` is ignored by Git.

### 2. Create the database

For a new database:

```bash
npx wrangler d1 execute digital_product_bot --remote --file=./schema.sql
```

For an existing deployment, apply the ordered files in `migrations/`, including `015_binance_pay.sql`, `016_restore_mobile_payments.sql`, and `017_redeem_codes.sql`.

### 3. Store secrets in Cloudflare

```bash
npx wrangler secret put TELEGRAM_BOT_TOKEN
npx wrangler secret put TELEGRAM_SECRET_TOKEN
npx wrangler secret put WEBHOOK_SECRET
npx wrangler secret put QUANTUM_VAULT_API_KEY
npx wrangler secret put BINANCE_API_KEY
npx wrangler secret put BINANCE_SECRET_KEY
```

The Binance API key should be read-only and limited to the access needed to read Binance Pay transaction history. Do not enable trading or withdrawals. Configure `BINANCE_PAY_ID` and `BINANCE_USDT_TO_BDT_RATE` in your untracked `wrangler.toml`.

### 4. Test and deploy

```bash
npm test
npx wrangler deploy --dry-run
npx wrangler deploy
```

Set the webhook after deployment:

```bash
TELEGRAM_BOT_TOKEN=your-token node scripts/set-webhook.mjs "https://YOUR-WORKER.workers.dev" "YOUR_WEBHOOK_SECRET"
```

## Payment verification

Add Balance offers Binance Pay, bKash, Nagad, and Upay, with Rocket shown as temporarily unavailable. Mobile-wallet payments are matched against private Firestore transaction records using the selected provider, entered amount, and transaction ID. Binance Pay queries successful incoming USDT transactions using the submitted Order ID. Both flows prevent reuse, show a temporary `Verifying...` message, and delete it when verification finishes.

## Security

Read [SECURITY.md](SECURITY.md) before deploying. Never commit credentials or production configuration. If a secret was pasted into chat, a screenshot, or Git, rotate it before using the deployment.
